java接口学习

Created2021-12-24|Updated2021-12-24
|Post View:








接口

调用方式

  • 接口中的属性默认静态，常量
  • 方法均为抽象方法，没有方法体
  • 静态方法可以有方法体，直接由接口名调用
  • 接口中还有default方法，所有实现了接口的类都继承这个方法

Static Final

  • static : 静态的 唯一

    • 属性 不属于对象，可以由类名直接调用 且只有一份
    • 方法 不属于对象，可以由类名直接调用 且只有一份
    • 类 ： 内部类
    • 代码块：类使用时执行

  • final : 最终的 不可变

    • 属性 ：必须初始化值 之后对象调用final 修饰的常量 不能修改值
    • 方法 ：方法不能被重写
    • 类: 不能被继承 String
  • 重写的方法： 子类以及子类对象优先调用
  • 接口的使用方法：
    • 创建一个类实现implements 接口
    • 必须重写接口中所有的抽象方法
  • 类： 抽象的
  • 类相对于对象抽象： 属性抽象 方法还是具体的>类是不完全抽象的
  • 接口：属性是具体的，方法抽象（抽象行为）
  • 类+接口： 实现对一类事物/一类行为进行抽象处理

为什么java中要有接口的存在？？

大概是为了软件设计人员更好的进行顶层的设计，不必深究其中的细节，这样子能更好的全局把握软件的开发!
